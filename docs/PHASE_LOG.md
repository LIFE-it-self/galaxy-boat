Notes:
- The first `RhythmBar` note arrives at `tMs = 1000` with `NOTE_TRAVEL_MS = 1500`, so its spawn offset is `-500`. The partial-travel branch handles this by computing `fraction = 500 / 1500 = 1/3`, spawning at `rightEdgeX - (1/3) * barSpan`, and tweening the remaining 1000ms to the hit zone. Without this, the first note would either be missing or misaligned.
- All `delayedCall` callbacks in the new minigames check `this.state === 'PLAY'` before touching anything. This is essential because once `win()` or `lose()` fires, the state flips away from `'PLAY'` and the result overlay runs for ~1.2s before the scene actually shuts down â€” any in-flight callbacks must no-op during that window or they'll touch destroyed objects and crash.
- `PipeSmoke` does NOT call `markRitualStep` itself. `BaseMinigame.win()` already does that automatically when the level config has `isRitual: true`, so calling it manually would push step 1 onto `ritualProgress` twice.
- `CokeDrinkGame` and `PipeSmoke` both use `this.input.on('pointerdown', ...)` AND `this.input.keyboard.on('keydown-SPACE', ...)`. Either input source advances the game â€” important for mobile-first.
- Trigger markers are re-rendered every time `OverworldScene` runs `create()`, which happens on every door transition (`scene.restart()`). Since `renderTriggerMarkers` reads `completedMinigames` fresh each time, dormant triggers stay dormant after the player leaves and re-enters the room.
- `BaseMinigame.lose()` does NOT call `markMinigameComplete`, so a failed `coke-drink` attempt leaves the trigger active and the player can re-step on it to retry. This is what makes the 5-fail hurricane test work naturally â€” repeated CokeDrink losses tick the failure counter without unblocking the trigger.
